# Jianying Boundaries

## Default Rule

Do not modify existing Jianying drafts by default. Draft changes need a loop that explicitly authorizes the target draft, operation, and validation method.

## Accepted Evidence Levels

Record validation evidence honestly:

- file-level generation check
- manifest/schema validation
- Jianying homepage visibility
- user-reported manual playback
- screenshot evidence
- screen recording evidence
- exported playback evidence

Do not upgrade user-reported playback into screenshot or recording evidence.

## Draft Positioning

Preview drafts are internal shooting previews and placeholder drafts unless a later loop proves final publish readiness.

For alpha workflow packages, any generated or imported preview should be treated as an internal preview until a later loop records stronger evidence and publish readiness.

## Packaging Boundary

Prefer seed-draft clone plus manifest validation over direct manual JSON editing. Jianying draft files should not be treated as ordinary JSON unless the project has already validated that specific write path.

## Replacement Boundary

Editable content, subtitle, voiceover, and audio plans must remain outside the draft as source-of-truth control files. Writing them into Jianying should be an export/import loop, not the only copy.
